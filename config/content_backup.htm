近端配置及管理

cd - 由母資料夾轉換至子資料夾的指令，cd(空格)子資料夾名。

dir - 查看資料夾中的檔案列表及相關資訊。

mkdir - 新增新資料夾，如有遇到因為需用特殊字元當名稱無法新增資料夾的情況，可利用 mkdir(空格)新資料夾名來建立新資料夾。

git clone - 將Github倉儲內容複製到隨身系統中。

ssh-keygen -t rsa -b 4096 -C "This is my key" - 建立成對的SSH鑰匙，並將公開的上傳至Github，此後即可利用SSH進行資料推送，""中可輸入指令以便辨識多組鑰匙。

隨身系統

隨身碟當中通常內設的格式為FAT32/exFAT，若要載入大型檔案(4GB以上)將會受阻，所以在要放入Ubuntu 18.04 vdi 檔案之前, 需先將隨身碟格式化為 NTFS，否則就算隨身碟內仍有許多檔案仍會無法執行成功。(格式化前記得先將資料備份!)

  • FAT32(不建議使用)

兼容性佳，安全度不高，單一文件大小不能超過4GB、名稱不能超過225個字元。

  • exFAT(FAT64，適用行動設備)

兼容性佳，最大可容納不超過256GB的單位文件。

  • NTFS(適用存放數據)

管理方便、安全性佳、容錯率高、穩定性高，屬於微軟專利其他使用者須藉由第三方才可使用。
